1. Tic-Tac-Toe Endgame Classification¶

For introduction and rules of Tic-Tac-Toe, see Wiki page.

We will use Tic-Tac-Toe Endgame Data Set from UCI machine learning repository. (See introduction here). This database encodes the complete set of possible board configurations at the end of tic-tac-toe games, where "x" is assumed to have played first. The target concept is "win for x" (i.e., true when "x" has one of 8 possible ways to create a "three-in-a-row").

The dataset has 9 attributes, each indicating the status of each squre. ('x' if "x" is placed, 'o' if "o" is placed and 'b' if blank). Examples of the dataset can be seen here:

Post-processing¶

To get these features and labels fit into learning models, we need to convert them into integer or boolean variables. For features with $k$ possible values, a common trick is one-hot encoding (dummy variables). Suppose feature $X$ can take $k$ possible values, we encode it into a $k$-dimensional boolean vector where there is a one at location $i$ if $X = 1$ and zeros elsewhere. Suppose $X$ can take $5$ possible values and $X = 3$. Then

$$ Enc(X) = (0, 0, 1, 0, 0) $$

We do this for each feature with $k = 3$ and convert labels to $\{0, 1\}$. Note that for one hot encoding, we can always drop the first dimension since if all others are zeros, we know it has to be one.

Next we randomly pick $70\%$ of the data to be our training set and the remaining for testing.

After this, each training example has $18$ features. Training set looks like the following:

Using information gain as you splitting criterion, set the maximum depth from 1 to 12. Plot the training and testing error with respect to each maximum depth. Justify your plot.

Err_Train = np.zeros(12)

Explanation for Problem 1 ¶

In the plot shown above, as we increase the maximum depth of the tree, the error goes down from around 30% to around 5%. This trend is intuitive because as we add more decisions (increasing depth), we expect the classifier to be more accurate. Comparing the testing and training plots, we see that the testing plot is always less accurate than the training plot. This is expected as the classifier has not seen the testing set yet and will predict at a lower accuracy than the training set.

2. Chess(King-Rook vs. King) Endgame Classification¶

For introduction and rules of Chess, see Wiki page.

We will use Chess(King-Rook vs. King) Data Set from UCI machine learning repository. (See introduction here). This database has 28056 possible instances of chess endgame situations where the white has a king and a rook and the black has only a king. The goal is to determine what is the minimum depth for the white to win.

The dataset has 6 attributes. Each of them can take 8 values, listed as following:

  1. White King file (column a - h)
  2. White King rank (row 1 - 8)
  3. White Rook file
  4. White Rook rank
  5. Black King file
  6. Black King rank

And the label is the least number of steps that the white must use to win. (draw if more than 16). The following is how the data set looks like.

Explanation for Problem 4¶

The maximum training accuracy is achieved at a maximum depth of approximately 30. The maximum testing accuracy is achieved around 30 as well, although the accuracy has not increased by a lot when compared to the initial depth of 20. In this situation, we can deduce that the model cannot generalize and perform well on the testing set at a maximum depth beyond 20. The model is overfitted and cannot perform well on data it has not yet seen. A fix for this would be to decrease the maximum depth.

Problem 5¶

Let's take a step further towards real AI applications. For the same game set-up, suppose you have a perfect decision tree which can tell you the minimum number of moves the white need to win. Given any instance, can you tell us which move is the optimal move for the white?

Yes, you can determine the optimal move for white. Given a position, compute its current minimum number of moves to win. Then, generate a set of candidate moves and update the current position by going through each of the candidate moves. Then, compute the new number of minimum moves to win using the tree. The optimal move is the one which results in a position with minimum moves that is one less than the original minimum moves. We can keep performing this until we win.
