In this assignment, we are going to revisit the chess(King-Rook vs. King) Endgame Classification problem we saw in the first assignment. Recall that using decision trees, we couldn't get good testing accuracy (around 55%). We will try to improve this using ensemble methods.

Chess(King-Rook vs. King) Endgame Classification¶

For introduction and rules of Chess, see Wiki page.

We will use Chess(King-Rook vs. King) Data Set from UCI machine learning repository. (See introduction here). This database has 28056 possible instances of chess endgame situations where the white has a king and a rook and the black has only a king. The goal is to determine what is the minimum depth for the white to win.

The dataset has 6 attributes. Each of them can take 8 values, listed as following:

  1. White King file (column a - h)
  2. White King rank (row 1 - 8)
  3. White Rook file
  4. White Rook rank
  5. Black King file
  6. Black King rank

And the label/class is the least number of steps that white must use to win. (draw if more than 16). The following is how the data set looks like.

2. Adaboost.¶

Decision trees with small maximum depth won't give us good performance because of limited complexity. In this problem, we use adaboost algorithm to reduce the bias of the model and hopefully this will give us better performance. from sklearn.ensemble import AdaBoostClassifier
n_max = 12
Err_Train = np.zeros(n_max)
Err_Test = np.zeros(n_max)
indices = 2**np.array(range(0,n_max))
depth = [10,20,50,100]

    for i in range(0,n_max):
        clf = AdaBoostClassifier(base_estimator=DecisionTreeClassifier(criterion='entropy',max_depth=depth[j]), n_estimators=indices[i]).fit(X_train, y_train)
        y_pred_train = clf.predict(X_train)
        y_pred_test = clf.predict(X_test)

        Err_Test[i] = hamming_loss(y_test, y_pred_test)

Explanation for Problem 2 ¶

-At a depth of 10, as we increase the number of base estimators, the training error and testing error both increase slightly then decrease significantly. The training error reaches a minimum of 0.1 near a base estimator count of 500. The testing error reaches a minimum of around 0.4. As expected, the training error is lower than the testing error. With low depth and high number of iterations, the Adaboost estimator is less likely to overfit on the training data. For the training error, we cannot reach an error of 0 at the given base estimator counts.

-At a depth of 20, we see a rapid decline in training error from 0.1 to 0 as we increase the number of base estimators beyond 2. For testing error, we see an increase then a decrease in testing error from above 0.5 to around 0.35. This is likely the optimal configuration, as the training error reaches a minimum of zero at a base estimator count after 2, and testing error reaches a minimum of around 0.35.

-At a depth of 50, we see a much different behavior for both the training and testing error. Both plots are almost completely flat. The training error is zero across all base estimator counts and the testing error is approximately 0.475. This is likely due to overfitting due to the high depth of the tree base classifier. With a depth of 50, the model is completely tuned to the training data and cannot generalize well on the testing data. Thus, we do not see any decline in testing error with increasing number of iterations. We can conclude that boosting works well for weaker base classifiers.

-At a depth of 100, the argument is the same as that of a depth of 50. There is significant overfitting and cannot generalize on the testing data, resulting in a relatively high testing error.
